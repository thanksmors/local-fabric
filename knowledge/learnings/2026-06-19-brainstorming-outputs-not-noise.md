## Learning

Brainstorming is most valuable as a user-facing decision process. OKF memory should usually capture the durable output — selected decision, implementation plan, research note, or reusable learning — instead of preserving every raw idea.

## Applies To

- Ambiguous architecture, product, process, and workflow choices.
- Multi-step implementation planning.
- Agent handoff and future context reconstruction.

## Operational Rule

Use numbered options with a recommended choice during the conversation. Save a durable artifact only when it changes repo behavior, guides future work, supports handoff, or records a meaningful decision.

## Anti-Pattern

Creating `knowledge/brainstorms/` full of raw exploratory transcripts before there is a current consumer.
